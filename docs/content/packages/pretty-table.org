=pretty-table= provides two table implementations:

- =Table(N)= / =TableBuilder(N)= — column count known at *comptime*. Supports per-cell styling (color, bold, italic), column alignment, and column spanning.
- =DynTable= — column count known at *runtime*. Plain text only, with built-in cell truncation and =max_width= support.

** Which one to choose?

| Scenario | Use |
|---|---|
| Column count is a literal (e.g. always 3 columns) | =Table(N)= |
| Rows are built dynamically but columns are fixed | =TableBuilder(N)= |
| Column count depends on input data (CSV, DB query) | =DynTable= |
| Need per-cell color, bold, alignment, hspan | =Table(N)= / =TableBuilder(N)= |
| Need auto-truncation to fit terminal width | =DynTable= |

The key constraint is Zig's comptime generics: =Table(N)= requires =N= to be a compile-time constant.
If the column count comes from runtime data (e.g. parsing a CSV header), =DynTable= is the only option.

* Features (Table / TableBuilder)
- Box-drawing character sets: =ascii=, =box=, =dos=.
- Per-cell styling: foreground/background color, bold, italic.
- Column spanning (=hspan=): a cell can span multiple columns.
- Per-column text alignment: =left=, =right=, =center=.
- Configurable cell padding and row separators.

* Features (DynTable)
- Same border styles: =ascii=, =box=, =dos=.
- Configurable padding.
- Auto-detects terminal width; cells are truncated with =…= ellipsis to fit. Override with =max_width=.

* Usage
See [[https://github.com/jiacai2050/zigcli/blob/main/examples/pretty-table-demo.zig][pretty-table-demo.zig]].

** Basic table

#+begin_src zig
const Table = @import("pretty-table").Table;
const Cell = @import("pretty-table").Cell;

const t = Table(2){
    .header = [_]Cell{ Cell.init("Language"), Cell.init("Files") },
    .rows = &[_][2]Cell{
        .{ Cell.init("Zig"), Cell.init("3") },
        .{ Cell.init("Python"), Cell.init("2") },
    },
    .footer = [2]Cell{ Cell.init("Total"), Cell.init("5") },
    .mode = .box,
    .padding = 1,
};

const out = std.fs.File.stdout();
var buf: [4096]u8 = undefined;
var writer = out.writer(&buf);
try writer.interface.print("{f}", .{t});
try writer.interface.flush();
#+end_src

#+begin_example
┌──────────┬───────┐
│ Language │ Files │
├──────────┼───────┤
│ Zig      │ 3     │
│ Python   │ 2     │
├──────────┼───────┤
│ Total    │ 5     │
└──────────┴───────┘
#+end_example

** Column spanning

#+begin_src zig
const t = Table(4){
    .header = [_]Cell{
        Cell.init("Name"), Cell.init("Q1"), Cell.init("Q2"), Cell.init("Q3"),
    },
    .rows = &[_][4]Cell{
        .{ Cell.init("Alice"), Cell.init("90"), Cell.init("85"), Cell.init("92") },
        .{
            Cell.init("Bob"),
            Cell.init("N/A (on leave)").withHspan(3),
            Cell.span(),
            Cell.span(),

    .mode = .box,
    .padding = 1,
};

** TableBuilder (runtime rows)

When the number of rows is not known at comptime, use =TableBuilder(N)=.
Column count is still comptime-fixed for type safety.

#+begin_src zig
const pt = @import("pretty-table");

var t = pt.TableBuilder(3).init(allocator, .{
    .mode = .box,
    .padding = 1,
    .column_align = .{ .left, .right, .right },
});
defer t.deinit();

t.setHeader(.{ "Name", "Score", "Rank" });
t.setFooter(.{ "Total", "", "—" });

// String literals are auto-wrapped in Cell.init().
try t.addRow(.{ "Alice", "9800", "1" });
try t.addRow(.{ "Bob", "7500", "2" });

// Use addRowCells for per-cell styling.
try t.addRowCells(.{
    pt.Cell.init("Charlie").withBold(),
    pt.Cell.init("9900").withFg(.green),
    pt.Cell.init("1").withFg(.green),
});

try writer.interface.print("{f}", .{t});
try writer.interface.flush();
#+end_src

** DynTable (runtime columns)

When the column count is only known at runtime, use =DynTable=.
It provides a builder API with =addRow= (string slices) and =addRowCells= (=Cell= values for styling).

#+begin_src zig
const pt = @import("pretty-table");

var table = pt.DynTable.init(allocator, 3);
defer table.deinit();
table.mode = .box;
table.padding = 1;

try table.setHeader(&.{ "Name", "Age", "City" });
try table.addRow(&.{ "Alice", "30", "Beijing" });
try table.addRow(&.{ "Bob", "25", "New York" });
try table.render(writer);
#+end_src

#+begin_example
┌───────┬─────┬──────────┐
│ Name  │ Age │ City     │
├───────┼─────┼──────────┤
│ Alice │ 30  │ Beijing  │
│ Bob   │ 25  │ New York │
└───────┴─────┴──────────┘
#+end_example

Use =addRowCells= for per-cell styling:

#+begin_src zig
try table.addRowCells(&.{
    pt.Cell.init("Charlie").withBold(),
    pt.Cell.init("35"),
    pt.Cell.init("London").withFg(.green),
});
#+end_src

*** Truncation with max_width

=DynTable= auto-detects the terminal width and truncates cells to fit.
In most cases no configuration is needed — just leave =max_width= at its default (0).

To override, set =max_width= explicitly:

#+begin_src zig
table.max_width = 80; // 0 (default) = auto-detect terminal.
#+end_src
